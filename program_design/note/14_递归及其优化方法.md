## 递归

与动态规划相反，递归是自顶向下的过程

递归一般是通过减治、分治的思想来解决问题  
还有就是和深度优先搜索，决策树，回溯搜索的感觉时都可以考虑使用递归  

但一般在使用过程中容易出现大量的重复计算，导致栈溢出  

### 优化方法

1. 尾递归优化(Tail Call Optimization, TCO)

如果一个函数的最后一个动作是调用自身，且没有后续的算术操作，编译器可以将其优化为类似 goto 的跳转，从而不再增加新的栈帧
- 普通递归：`return n * factorial(n - 1);`
- 尾递归：`return factorial(n - 1, acc * n);`

是否会进行 TCO 与编译器相关

2. 记忆化搜索

针对具有“重叠子问题”特征的递归（如斐波那契数列），通过空间换时间，将已经计算过的结果存储，可以大量减少时间浪费

这是一种比较好用的方法，对绝大多数情况适用

```C++
long long fib(int n, vector<long long>& memo) {
    if (n <= 1) {
        return n;
    }
    if (memo[n] != -1) {
        return memo[n];
    }
    return memo[n] = fib(n - 1, memo) + fib(n - 2, memo);
}
```

3. 显示栈模拟与动态规划

如果递归逻辑太复杂(如深度优先搜索 DFS)，又担心系统栈(System Stack)溢出，可以手动维护一个 std::stack

因为系统栈空间通常只有几 MB，而堆空间(heap)只受限于物理内存 

类似的还有改为自底向上的 dp
