## 浮点数精度问题

在 C++ 中，由于浮点数底层采用二进制算术标准，容易引发精度问题

常见情况有：
1. 浮点数进行相等比较(==)
2. 极大的数与极小的数进行相加(大数会吞掉小数)
3. 两个非常接近的数进行相减(有效数位丢失)

最好的解决措施是加上 `eps` 误差，以及善用 `fabs()`   

```C++
double x = 0.1 + 0.2;
double eps = 1e-9
if (fabs(x - 0.3) < eps) {
    ...
}
```

对于 `eps = 1e-9` 是有讲究的，从结果上来说是最优的，但对于 `float` 来说，`eps = 1e-6`  

反正一般而言，`eps` 至少比输出精度多 2~3 位即可
