### 打开文件

open() 函数是三个文件流对象的一个成员函数

```C++
void open(const char *filename, ios::openmode mode);  // f.open(文件名, 模式)
```

|模式标志|描述|
|---|---|
|`ios::in`|读取，允许从流中读取数据|
|`ios::out`|写入，允许向流中写入数据|
|`ios::trunc`|截断，如果文件已存在，打开时清空文件内容truncate|
|`ios::app`|追加，所有写入操作都会强制跳转到文件末尾，append|
|`ios::ate`|末尾，打开文件后立即将文件指针移动到末尾，at end|
|`ios::binary`|以二进制打开|

默认值提醒  
- `ifstream`: 默认为 `ios::in`
- `ofstream`: 默认为 `ios::out | ios::trunc`
- `fstream`: 默认为 `ios::in | ios::out`

值得注意的是，ate 只是在打开那一刻把光标移到最后，仍然可以通过 seekg() 或 seekp() 把移动光标修改文件中间的数据，而 app 是一种强制行为，只要调用 write 就会自动把光标移动到文件末尾，保证不覆盖之前的数据   

```C++
ifstream fin("date.txt");
if (!fin.is_open()) {
    cerr << "Error: file open failed!" << endl;
}
fin.close()
```

### 文件读取

1. 格式化读取：`>>`
   - 自动跳过空格、制表符和换行符
   - 自动处理数据类型转换，如字符串转成 int
2. 按行读取：`getline()`
   - 读取到换行符为止，保留空格
   - 自动添加 `\0`
3. 按字符或数据块读取：`get()`, `read()`

其中 `getline` 常搭配 `str.find()` 和 `str.substr()` 使用  

比如 info.txt 文档格式为: name age id

```C++
ifstream fin("info.txt");
if (!fin.is_open()) {
    cerr << "Error" << endl;
}
string line;
while (getline(fin, line)) {
    string name, id;
    int age;
    if (line.empty()) {  // 留意检查空行
        continue;
    }
    size_t pos = line.find(" ");   // string::npos 非常大，用 size_t 防止溢出
    if (pos != string::npos) {
        name = line.substr(0, pos);
        size_t pos2 = line.find(" ", pos + 1);
        if (pos2 != string::npos) {
            age = stoi(line.substr(pos + 1， pos2 - pos));  // std::stoi()
            id = line.substr(pos2 + 1);
        }
    }
}
fin.close();
```

要实现相同的功能，也可以利用 `stringstream` 更加智能便捷，在 `sstream` 头文件中  

```C++
stringstream ss(line);
string name, id;
int age;
ss >> name >> age >> id;
```

`stringstream` 的优势
- 自动类型转换
- 智能处理空白符，即使空格数量不确定
- 配合 `getline()` 分隔符参数更神

如果 info.txt 文档格式变为：name-age-id

```C++
ifstream fin("info.txt");
if (!fin.is_open()) {
    cerr << "Error" << endl;
}
string line;
while (getlie(fin, line)) {
    if (line.empty()) {
        continue;
    }
    stringstream ss(line);
    string name_str, age_str, id_str;
    getline(ss, name_str, '-');
    getline(ss, age_str, '-');
    getline(ss, id_str, '-');
    // 再进行类型转换即可
}
fin.close();
```

如果读取内容较大，避免在循环中频繁创建 stringstream，可以使用 `clear()` 继续复用

```C++
stringstream ss;
while (getline(file, line)) {
    ss.str(line);   // 清空先前内容，并赋值
    ss.clear();     // 重置状态
}
```

更一般的，实现逐单词读取  

```C++
string line;
while (getline(fin, line)) {
    if (line.empty()) continue;
    string word;
    for (char c : line) {
        if (isalpha(c)) {
            word += c;
            ...
        } else {
            if (!word.empty()) {
                word.clear();
            }
        }
    }
    if (!word.empty()) {
        ...
    }
}
```

如果要实现更加精细的读取，比如按字符读取，需要使用 `get()`，能够读取单个字符（包括空格和换行）   

```C++
char ch;
while (file.get(ch)) {
    if (ch != '\n') {
        cout << "读到：" << ch << endl;
    }
}
```

此外还有 `peek()`，同样是读取单个字符，但它不会将字符从流中提取出来，起到预览作用，还有 `putback()`，将字符放回流中

在日常的开发中，更常使用 `read()` 按照字节数将文件原始数据拷贝到内存缓冲区中，速度最快，一般用于加载二进制文件  

```C++
istream& read(char_type* buffer, streamsize n);
```

```C++
ifstream file(filename, ios::binary);
char buffer[1024];
while (file.read(buffer, sizeof(buffer))) {
    streamsize bytes = file.gcount();
    // 处理这 bytes 字节...
}
file.close();
```

可以利用 `reinterpret_cast` 将文件中的数据读取到结构体中，达到快速存取  

```C++
struct Player {
    int id;
    string name;
    double health;
    int level;
};

ifstream flie("player_save.dat", ios::binary);
if (!file.is_open()) {

    return 0;
}
Player p;
file.read(reinterpret_cast<char*>(&p), sizeof(Player));
if (file.gcount() == sizeof(Player)) {
    cout << p.name << ... 
}
file.close();
```

### 文件的写入

1. 与读取相似，注意打开模式  
2. 无论是 `<<` 还是 `getline()` 都要手动换行
3. 可以使用 iomanip

```C++
ofstream fout("config.txt");
if (!fout.is_open()) {
    cerr << "Error" << endl;
}
fout << "ID: " << 1001 << "\n";
fout << "Balance: " << 125.50 << endl;
fout.close();
```
 
```C++
ofstream fout("user.txt");
string input;
while (getline(cin, input) && input != "q") {
    fout << input << endl;
}
fout.close();
```

另一个值得注意的是：在交互中，使用 `cin` 会读取 `\n` 前的内容，并把 `\n` 留在流中，而 `getline()` 会丢弃 `\n`
