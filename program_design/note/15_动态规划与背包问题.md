## 动态规划(Dynamic Programing, DP)

参考：[](https://zhuanlan.zhihu.com/p/93857890)

动态规划是通过解决子问题，将结果存储起来，从而避免重复计算，最终推导出全局最优解的方法，与递归相反，是自底向上  

主要是看能否通过局部最优推到出全局最优，以及计算出的状态确定后，是否会受后续决策影响

一般分为三个步骤
1. 确定状态定义(State)：最难的一步，需要定义好 dp[i][j] 代表什么
2. 推导状态转移方程(Transition)
3. 确定边界条件和初始化(Initialization)

常见的利用 dp 解决的问题有
1. 背包问题
2. 序列问题
3. 路径问题
4. 区间问题

## 背包问题

### 01背包

> 有 $N$ 件物品和一个容量为 $V$ 的背包。第 $i$ 件物品的重量是 $w_i$，价值是 $v_i$。**每种物品仅有一件**。求将哪些物品装入背包可使价值总和最大。

我们可以定义状态    
- dp[i][j]表示将前i件物品装进限重为j的背包可以获得的最大价值, 0<=i<=N, 0<=j<=W

那么对于对于每一件物品，都有两种选择
- 不装入第i件物品，即dp[i−1][j]
- 装入第i件物品，如果能装下的话，即dp[i−1][j−w[i]] + v[i]

由此得出状态转移方程 $dp[i][j] = max(dp[i−1][j], dp[i−1][j−w[i]]+v[i])  j >= w[i]$

```C++
vector<vector<int>> dp(N + 1, vector<int>(V + 1， 0));
for (int i = 1; i <= N; i++) {
    for (int j = 1; j <= V; j++) {
        if (weight[i] > j) {
            dp[i][j] = dp[i - 1][j];
        } else {
            dp[i][j] = max(dp[i - 1][j], dp[i - 1][j - weight[i]] + val[i]);
        }
    }
}
```

但通过对状态转移方程的观察，dp[i][j]的值只与dp[i-1][0,...,j-1]有关，所以我们可以采用动态规划常用的方法（滚动数组）对空间进行优化

```C++
vector<int> dp(V + 1, 0);
for (int i = 1; i <= N; i++) {
    for (int j = W; j >= weight[i]; j--) {
        dp[j] = max(dp[j], dp[j - weight[i]] + val[i]);
    }
}
```

值得注意的是选择逆序遍历，这是为了保证计算前 $i$ 个物品的结果时，利用的是前 $i-1$ 个物品的数据，因为会用到 dp[j - weight[i]]，可以画图理解  

### 完全背包

> 有 $N$ 件物品和一个容量为 $V$ 的背包。第 $i$ 件物品重量为 $w_i$，价值为 $v_i$。**每种物品都有无限件可用**。求价值最大化的装包方案

总体与 01 背包相似，同样考虑
- 不装入第i种物品，即dp[i−1][j]
- 装入第i种物品，由于物品可以无限选，所以转移到dp[i][j−w[i]]，这与 01 背包不同

转移方程为 $dp[i][j] = max(dp[i−1][j], dp[i][j−w[i]]+v[i]) j >= w[i]$

```C++
vector<vector<int>> dp(N + 1, vector<int>(V + 1, 0));
for (int i = 1; i <= N; i++) {
    for (int j = 1; j <= V; j++) {
        if (weight[i] > j) {
            dp[i][j] = dp[i - 1][j];
        } else {
            dp[i][j] = max(dp[i - 1][j], dp[i][j - weight[i]] + val[i]);
        }
    }
}
```

空间优化版，注意是正向遍历，画图理解过程后显然，这是由状态方程决定的

```C++
for (int i = 1; i <= N; i++) {
    for (int j = weight[i]; j <= V; j++) {
        dp[j] = max(dp[j - 1], dp[j - weight[i]] + val[i]);
    }
}
```

### 多重背包

> 有 $N$ 件物品和一个容量为 $V$ 的背包。第 $i$ 件物品重量为 $w_i$，价值为 $v_i$，**且数量限定为 $n_i$ 件**。求价值最大化的方案

这里需要多分析一遍第 i 个物品放入的数量，也很简单

```C++
vector<vector<int>> dp(N + 1, vector<int>(V + 1, 0));
for (int i = 1; i <= N; i++) {
    for (int j = 1; j <= V; j++) {
        if (weight[i] > j) {
            dp[i][j] = dp[i - 1][j];
        } else {
            for (int k = 0; k <= j / weight[i] && k <= num[i]; k++) {
                dp[i][j] = max(dp[i][j], dp[i - 1][j - k * weight[i]] + k * val[i]);
            }
        }
    }
}
```

空间优化与 01 背包相同，采用逆向遍历  

实际上，多重背包可以转化为 01 背包，只需要按将同一物品分成不同组，编码成能表示任意数量即可，比如二进制拆分 

```C++
vector<int> new_weight;
vector<int> new_val;
for (int i = 1; i <= N; i++) {
    int count = num[i];
    int k = 1;
    while (k <= count) {
        new_weight.push_back(k * weight[i]);
        new_val.push_back(k * val[i]);
        count -= k;
        k *= 2;
    }
    if (cnt > 0) {
        new_weight.push_back(count * weight[i]);
        new_val.push_back(count * val[i]);
    }
}
```

然后对新数据进行 01 背包即可

### 分组背包

> 有 $N$ 组物品和一个容量为 $V$ 的背包。**每组物品内有若干件，但每组最多只能选一件物品**。求总价值最大的方案

同样与 01 背包和 多重背包类似，对每一组内取个 max 即可

### 其他问题

1. 恰好装满问题，与 01 背包思路一致，修改初始化逻辑
2. 方案总数问题，$dp[i][j] = sum(dp[i−1][j], dp[i][j−w[i]]) j >= w[i]$
3. 最优方案，再用一个数组G[i][j]来记录方案

```C++
// T: target 目标值
vector<bool> dp(T + 1, 0);
dp[0] = 1;
for (int i = 1; i <= N; i++) {
    for (int j = T; j >= weight[i]; j--) {
        dp[j] = dp[j] || dp[j - weight[i]];
    }
    if (dp[T]) {
        return 1;
    }
}
```

## 约束条件

![](../png/15_1.png)

*对于100%的数据，1 <= n <= 100,000，1 <= k <= n，0 <= a[i] <= 1,000,000,000*

这里增加的限制条件为不能连续 k 个，采用传统的 dp 行不通了，我们考虑**增加维度**
- dp[i][k]表示选取第 i 个物品，且这是连续第 k 个时的最大总喜爱值

```C++
vector<vector<ll>> dp(n + 1, vector<ll>(k + 1, 0));
for (int i = 1; i <= n; i++) {
    ll max_prev = 0;
    for (int j = 0; j <= k; j++) {
        max_prev = max(max_prev, dp[i - 1][j]);
    }
    dp[i][0] = max_prev;
    for (int j = 1; j <= k; j++) {
        dp[i][j] = dp[i - 1][j - 1] + val[i];
    }
}
```

此处还能优化空间，但极端情况下还是会超时
